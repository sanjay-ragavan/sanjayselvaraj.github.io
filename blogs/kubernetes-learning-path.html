My Kubernetes Learning Path

A running log of what I've learned so far, and the roadmap I'm using to get to production-grade, SRE-level depth.

I've been building up Kubernetes depth hands-on rather than just reading about it — running a real multi-node cluster locally, breaking things, and working through the stack layer by layer. This post is partly a log for myself and partly a map for anyone else on the same path. I'll keep updating it as I go.

What I've learned so far

1. Kubernetes fundamentals

  • What Kubernetes is
  • Cluster architecture
  • Control plane vs. worker nodes
  • Pods
  • Deployments
  • ReplicaSets
  • Services (basic understanding)
  • Namespaces

2. Kind (Kubernetes in Docker)

  • Installed Docker
  • Installed Kind
  • Created a multi-node Kind cluster (sre-lab)
  • Viewed cluster information
  • Switched contexts
  • Deleted and recreated clusters

3. kubectl

  • kubectl get
  • kubectl describe
  • kubectl logs
  • kubectl exec
  • kubectl apply
  • kubectl delete
  • kubectl scale
  • kubectl top
  • Viewing namespaces, nodes, and pods

4. YAML manifests

  • Pod manifest
  • Deployment manifest
  • Service manifest
  • Namespace manifest

5. Deployments

  • Creating Deployments
  • Scaling replicas
  • Updating images
  • Rolling updates
  • Rollbacks (basic)

6. Namespaces

  • Creating namespaces
  • Deploying applications into different namespaces
  • Listing namespace resources

7. ConfigMaps

  • Creating ConfigMaps
  • Using ConfigMaps in Pods
  • Environment variables from ConfigMaps

8. Secrets

  • Creating Opaque Secrets
  • Mounting Secrets

9. Storage

  • Persistent Volumes (PV)

  • Local Path StorageClass (Kind)

10. Resource monitoring

  • Installed Metrics Server

11. Helm
