Kubernetes Roadmap: From Fundamentals to Production-Grade

A complete, level-by-level map of Kubernetes — from the basics to production and advanced topics. Use it as a checklist for your own learning.

Kubernetes has a famously steep learning curve — not because any single concept is hard on its own, but because the topics span such different layers, from the basics of pods and deployments all the way to service mesh and multi-cluster operations.

This roadmap lays out that whole surface area, organized by level, so you can work through it in order or jump to whatever you're missing. It's aimed at anyone learning Kubernetes seriously, especially if you're heading toward an SRE or DevOps role and need production-relevant depth, not just enough to deploy a demo app.

The fastest way through any of this is hands-on: spin up a local cluster with Kind, Minikube, or k3d, and run every example yourself with kubectl rather than just reading about it.

Beginner

The fundamentals: what Kubernetes is, how a cluster fits together, and the building blocks you'll touch in every cluster you ever work with.

  • What Kubernetes is, and the problem it solves
  • Cluster architecture — control plane vs. worker nodes
  • Pods, Deployments, and ReplicaSets
  • Services (basic pod-to-pod networking)
  • Namespaces
  • Core kubectl commands: get, describe, logs, exec, apply, delete, scale, top
  • Writing YAML manifests for Pods, Deployments, Services, and Namespaces
  • ConfigMaps and Secrets — getting configuration and sensitive data into Pods

Intermediate

Once the basics are solid, this is where you learn to run real workloads reliably — not just a single web server — and let Kubernetes self-heal instead of babysitting it.

  • Persistent Volumes & Persistent Volume Claims
  • StorageClasses
  • StatefulSets, for stateful apps like databases
  • DaemonSets, for one pod per node
  • Jobs & CronJobs — run-to-completion and scheduled workloads
  • Init Containers & multi-container Pods
  • Liveness, Readiness & Startup Probes
  • Resource Requests & Limits
  • Horizontal Pod Autoscaler (HPA) & Vertical Pod Autoscaler (VPA)

Networking

How traffic actually reaches your Pods, both inside and outside the cluster.

  • Service types: ClusterIP, NodePort, LoadBalancer, ExternalName
  • Headless Services
  • DNS & CoreDNS
  • Ingress & Ingress Controllers
  • Network Policies — controlling pod-to-pod traffic

Security

Who can do what, and how workloads stay isolated from each other.

  • RBAC: Roles, ClusterRoles, RoleBindings, ClusterRoleBindings
  • Service Accounts
  • Pod Security Standards
  • Security Contexts
  • Admission Controllers

Storage (advanced)

Beyond basic PV/PVC: how storage actually gets provisioned and protected at scale.

  • CSI Drivers
  • Dynamic Provisioning
  • Volume Snapshots
  • Access modes: ReadWriteOnce, ReadOnlyMany, ReadWriteMany

Scheduling

Controlling exactly where Pods run, and why.

  • Labels & Selectors
  • Taints & Tolerations
  • Node Affinity, Pod Affinity & Anti-Affinity
  • Priority Classes

Observability

Knowing what's actually happening inside your cluster.

  • Metrics Server, Prometheus & Grafana
  • Alertmanager
  • Logging: Fluent Bit, Loki, Elasticsearch, Kibana
  • Distributed tracing with OpenTelemetry

Package management

Installing and managing applications without hand-writing every manifest.

  • Helm — installing, upgrading, and uninstalling releases
  • Helm charts & values files
  • Chart development & dependencies

Production Kubernetes

The operational concerns that only matter once real users depend on your cluster.

  • Rolling Updates & Rollbacks
  • Pod Disruption Budgets
  • Resource Quotas & LimitRanges
  • Backup & restore, etcd
  • High availability
  • Cluster upgrades

GitOps & CI/CD

Deploying changes to the cluster safely and repeatably.

  • GitOps concepts
  • Argo CD
  • Jenkins + Kubernetes
  • GitHub Actions + Kubernetes
  • Image automation

Cloud Kubernetes

Running on a managed provider instead of operating your own control plane.

  • Amazon EKS, Azure AKS, Google GKE
  • Cloud load balancers & IAM integration
  • Autoscaling & managed node groups

Advanced

Extending Kubernetes itself, for when you've outgrown the built-in resource types.

  • Custom Resource Definitions (CRDs) & Operators
  • Admission Webhooks
  • API Aggregation
  • Service mesh: Istio, Linkerd
  • Kustomize
  • Helm chart development
  • Multi-cluster Kubernetes

If you're aiming for an SRE or DevOps role

Once the beginner fundamentals are solid, this is a practical order for the rest — it tracks closely with what actually shows up in production Kubernetes environments:

  1. StatefulSets
  2. Jobs & CronJobs
  3. Liveness, Readiness & Startup Probes
  4. Resource Requests & Limits
  5. RBAC
  6. Ingress & Ingress Controller
  7. HPA & Cluster Autoscaler
  8. Taints, Tolerations & Affinity
  9. Network Policies
  10. Kustomize
  11. CRDs & Operators
  12. Service Mesh (Istio)

Treat this as a living checklist: work through each level, get genuinely hands-on rather than just reading, and by the time you reach the advanced section you'll have the kind of depth that actually holds up in production — and in interviews.
